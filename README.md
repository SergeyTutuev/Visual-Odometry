# Visual Odometry — визуальная одометрия

Проект реализует **визуальную одометрию** (Visual Odometry, VO) для оценки траектории движения камеры по стереопарам изображений.  
Алгоритм использует **ORB** для детектирования ключевых точек, **стереосопоставление** для получения карты диспаратности и **минимизацию проекционной ошибки** для оценки перемещения камеры между кадрами.

---

## 🧠 Математическая модель

### 1. Триангуляция 3D-точек по стереопаре

Для точки с координатами $(x_l, y_l)$ в левом кадре и известной функции диспаратности $d(x_l, y_l)$ координаты соответствующей точки в правом изображении:

$$x_r = x_l - d(x_l, y_l), \quad y_r = y_l$$

Пусть $P_l$ и $P_r$ — проекционные матрицы левой и правой камер (размером $3 \times 4$), переводящие 3D-координаты в однородные координаты на растре.  
Для пары соответствующих точек $q_l = (x_l, y_l, 1)^T$ и $q_r = (x_r, y_r, 1)^T$ 3D-точка $Q = (X, Y, Z, 1)^T$ находится из системы:

$$
\begin{cases} 
q_l \times (P_l Q) = 0 \\ 
q_r \times (P_r Q) = 0 
\end{cases}
$$

### 2. Модель движения камеры

Преобразование между кадрами задаётся матрицей $T$ размера $4 \times 4$:

$$
T = \begin{pmatrix}
R & t \\
0^T & 1
\end{pmatrix}
$$

где:
- $R$ — матрица поворота $(3 \times 3)$,
- $t = (t_x, t_y, t_z)^T$ — вектор переноса.

### 3. Проекционная ошибка

Для пары 3D-точек $Q_1$ (в системе текущего кадра) и $Q_2$ (в системе следующего кадра) проекции имеют вид:

$$
\begin{cases} 
q_{1}^{\text{pred}} = \pi(P_l T Q_2) \\ 
q_{2}^{\text{pred}} = \pi(P_l T^{-1} Q_1)
\end{cases}
$$

где $\pi((x, y, z)^T) = (x/z,\ y/z,\ 1)^T$ — переход к однородным координатам.

Пусть $q_1 = f(I_1)$, $q_2 = f(I_2)$ — реальные координаты ключевых точек.  
Проекционная ошибка:

$$
E = \begin{pmatrix} q_{1}^{\text{pred}} - q_1 \\ q_{2}^{\text{pred}} - q_2 \end{pmatrix}
$$

Искомые параметры $R, t$ находятся минимизацией суммы квадратов ошибок:

$$
R, t = \arg\min_{R,t} \sum_{i=1}^{n} \| e_i \|^2, \quad e_i \in E
$$

## Алгоритмические особенности

- **Детектор и дескриптор:** ORB (Oriented FAST and Rotated BRIEF)
- **Сопоставление ключевых точек:** FLANN + фильтр по расстоянию (отношение 0.7)
- **Стереосопоставление:** алгоритм SGBM (Semi-Global Block Matching) для получения карты диспаратности
- **Оптимизация:** метод наименьших квадратов (least_squares из SciPy) с проверкой выбросов
- **Параллельная обработка:** многопоточное выполнение для повышения производительности
- **Отрисовка траектории:** интерактивный график в Plotly
